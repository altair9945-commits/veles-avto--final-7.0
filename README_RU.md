# VELES BOT v7.0 — AUTO PAYMENT + DIGITAL DELIVERY

## Что вошло

- Автоматическая проверка входящих TON / GRAM платежей через TON Center API.
- Сопоставление по кошельку, memo заказа и сумме TON.
- Автоматическая выдача следующего фото конкретной позиции витрины.
- Каждая позиция имеет отдельную очередь:
  - «Шоколад — 1 плитка — 4000 ₽»;
  - «Шоколад — 2 плитки — 8000 ₽».
  Они никогда не смешиваются, потому что склад привязан к `item_id`.
- Фото хранятся на серверах Telegram; в базе хранится `file_id`.
- Выдача строго по порядку.
- Повторная выдача одной фотографии исключена атомарной резервировкой.
- История: заказ, клиент, позиция, номер выданного фото.
- Низкий остаток — уведомление в группу заявок.
- Если фото закончились — новый заказ по позиции не создаётся.
- Ручное подтверждение оплаты админом тоже запускает выдачу.
- Кнопка клиента «Проверить оплату».
- Прямая кнопка открытия TON-кошелька с суммой и memo.
- Старые функции v5.3 сохранены: витрина, несколько типов, районы, статусы, backup, роли, отчёты.

## Важно перед запуском

Сначала в боте:
`/backup`

В Railway Variables добавь:

- `BOT_TOKEN`
- `ADMIN_PIN`
- `ADMIN_IDS`
- `DATA_DIR=/data`
- `TONCENTER_API_KEY`

Ключ TON Center создаётся через официальный `@toncenter` bot.

## Настройка

1. `/admin`
2. `💳 Оплата` — добавь TON / GRAM кошелёк.
3. `/admin → 🤖 Автооплата + выдача`
4. `💱 Указать курс TON/₽`
5. `📤 Загрузить фото`
6. Выбери **конкретную позицию**.
7. Отправляй фото подряд.
8. Нажми `✅ Завершить загрузку`.
9. Включи автооплату.

## Проверка

Сделай тестовую позицию и один реальный небольшой перевод.
В логах должно быть:

`VELES BOT v7.0 AUTO PAYMENT DIGITAL DELIVERY запущен.`

## Ограничение

Автоматическая проверка зависит от доступности TON Center API.
При недоступности API бот не выдаёт товар ошибочно: заказ остаётся ждать оплату,
админ может проверить и подтвердить вручную.
